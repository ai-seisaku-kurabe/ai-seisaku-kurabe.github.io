# 次の国政選挙の日程 — 取り決め

トップページに出す「次の国政選挙」と、選挙期間中の停止スイッチ（`election_mode`）は、
どちらも `tools/election_schedule.json` を正本とします。ここはその書き方の取り決めです。

## 1. 日付を書くのは人。機械は知らせるだけ

`koji`（公示日）と `vote`（投票日）を書き込めるのは**人だけ**です。
`agents/watch_election.py` は一次情報の変化を検知して知らせますが、
**日付を election_schedule.json に書き込みません**。

理由は2つあります。

1. 選挙期日を1日読み違えたまま公開すると、サイトが事実の誤りを載せます。
2. その日付は `election_mode` の切替にも使われるので、
   誤りが「公示後なのに集計が止まらない／公示前なのに止まる」という
   **法的リスクの誤作動**に直結します（公職選挙法第138条の3）。

機械が拾った日付をそのまま公開に通す設計にはしません。

## 2. 出典は一次情報だけ

`source` に入れてよいのは、**総務省・官報・中央選挙管理会・両院の公式資料**です。
新聞社・選挙情報サイト・Wikipedia は、日付を知る手がかりにはしても、
`source` には書きません（サイト全体の出典ルールと同じ）。

`vote` を書いたのに `source` が空だと、③検証班（`verify_content.py` の
`check_election_schedule()`）が FAIL を出して止めます。

## 3. 決まっていないときは「未定」と書く

期日が決まっていない間は `koji` / `vote` を `null` のままにします。
トップページは「期日未定」と表示し、かわりに **任期満了日**
（＝いつまでに必ず選挙があるか）と、その根拠になる法律の定めを示します。
**推測の日付を置きません**（憲法6条「データが無い項目は捏造せず空欄にし、理由を明記する」）。

## 4. 期日が告示されたときの手順

1. 一次情報（総務省の報道資料・官報）で、公示日と投票日を確認する。
2. `tools/election_schedule.json` の該当する選挙に `koji` / `vote` / `source` を書く。
3. `python agents/sync_election_mode.py` を実行する（必要なら `config.json` が書き換わる）。
4. `python build_site.py && python deploy_to_repo.py` で再生成する。
5. `python agents/verify_content.py` を FAIL 0 で通してから push する。

投票日が過ぎたら、次の選挙の枠に書き換えます（①収集班が「投票日が過ぎています」と知らせます）。

## 5. 停止スイッチについて

`agents/sync_election_mode.py` は、公示日から投票日までを「選挙期間」として
`config.json` の `election_mode` を切り替えます。やっているのは日付の比較だけで、
判断は含みません。日次のワークフロー（`.github/workflows/election.yml`）が実行します。

- 公示日だけ分かっていて投票日が未記入のときは、**期間中とみなして止めます**。
  止め忘れの方が取り返しがつかないためです。
- `config.json` のスイッチはクライアント側の分岐にすぎません。
  荒らしを確実に止める唯一の手段は、Firebase コンソールでルールを
  `allow create: if false` に書き換えることです（`about.html` に開示済み）。

## 6. ①収集班が見張っているもの

`agents/watch_election.py`（日次）。

- **期限**（通信なし）: 掲載中の任期満了日と公職選挙法第32条から逆算し、
  「この日を過ぎて期日が未記入なのはおかしい」を判定します。ここは見落としません。
- **参議院「参議院議員通常選挙一覧」**: 掲載している任期満了日をこの表と毎回照合し、
  新しい回次が現れたら知らせます。
- **総務省の報道資料・選挙の結果**: 選挙期日に関わる新しい見出しを知らせます。
  取りこぼさない側に倒してあるので空振りもあります。
  空振りの害は「CIが赤くなる」だけで、見落としの害より小さいためです。
