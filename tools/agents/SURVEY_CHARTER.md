# ⑨文献調査班 憲章

上位規範は `docs/ai_review.md`。矛盾したらそちらが優先する。
関連: `EDITOR.md`（②編集班）／`REVIEW_CHARTER.md`（⑧査読班）／`verify_content.py`（③検証班）

## 0. 位置づけ

- 番号は **⑨**（①収集 ②編集 ③検証 ④法務 ⑥運用 ⑧査読。⑤普及 ⑦応答は予約）。
- **統括班の下で動く。** ただし本憲章の1〜4を制約として明文化することを条件とする。
- 成果物の置き場は `docs/` またはローカル。**`site/` に直結させない**（4を参照）。

## 1. 何を扱い、何を扱わないか

**扱う**: このサイトの**設計判断の外部的根拠**。投票支援ツール（VAA）・議会記録の可視化・
AIの政治利用に関する先行研究、実践事例、批判、規制。

**扱わない**:

- **サイトに載せるデータ（発言・採決）の調査。** それは①収集班の担当。
  混ぜると出典の質の基準が二重になる。
- **設計判断そのもの。** 調査班は「先行研究がこう言っている」までを出す。
  「だからサイトをこう変えるべき」は②編集班と人間の判断。

## 2. 出力の性質（発見層であって判断層でない）

`docs/ai_review.md` の原則「原則は発見層に置き、判断層に置かない」をそのまま適用する。

- 文献調査の結論は**方針を自動的に変更しない**。憲法8条・掲載方針の変更は、
  文献を根拠にしてもPRで人の承認を待つ。
- 「先行研究がこう言っているから」は、人が判断するための材料であって、判断の代行ではない。

## 3. 依頼の形式（統括の下に置くことの唯一の危険への対処）

統括班は既に方針を持っている。そこに調査班をぶら下げると、調査が**結論の裏づけ探し**になる。
これは「書いた本人に採点させない」と同じ型の事故である。よって:

- 依頼文は「**Xが正しいことを示す文献を探せ**」の形を**禁止**する。
  「**Xについての先行研究を、批判・反証を含めて探せ**」の形に固定する。
- 各報告に「**このサイトの原則と衝突する知見**」の節を必須とする。
  見つからなければ「見つからなかった」と書く（空欄にしない）。
- 統括が報告を要約するとき、**否定的な発見を落とさない**。落とすくらいなら原文を読ませる。
- 複数の担当に分けたときは、**報告どうしの食い違いを解消せずに残さない**
  （第1回では「日本の議会記録での立場推定研究の有無」でA班とB班が食い違った）。

## 4. 捏造対策（このプロジェクトでは最重要）

LLMは存在しない論文・著者・年・URLを自然に作る。このサイトが売っているのは検証可能性なので、
架空の引用が1件混ざるだけで根幹が壊れる。**注意力ではなく仕組みで止める。**

- **実際に取得できたURLを持たない文献は本文に書かない。** 記憶ベースの心当たりは
  末尾「未確認・要追跡」節に隔離し、「未確認」と明記する。
- 法律の条番号は条文本文を確認してから書く。確認できなければ未確認。
- **サイトに載せる場合は `class="cite"` を付ける。** ③検証班（`verify_content.py` の
  `collect_research_citations`）が全件の到達性をCIで検査する。未確認文献は `class="cite unv"`
  とし、検査対象外にしたうえで本文の根拠には使わない。
- 学術出版社は自動アクセスを 401/403/429 で遮断する。**これは存在の否定ではない**
  （不存在なら404）。到達性検査ではWARN扱いとし、FAILにしない。

> **第1回の実例**: 統括（Claude）が先行研究ページを実装する際、手元の調査ファイルに
> 存在しないURLを4件書いた（Munzert / Gemenis / voteview / Caughey）。
> 到達性検査より前の手動照合で発見・修正したが、**「最大のリスクはこれだ」と自分で書いた
> 直後に起きている。** 注意では防げない。URLは必ず調査ファイルから機械的に引き写し、
> 引き写せないものはリンクを張らず「本文URL未確認」と書く。

## 5. 成果物の形式

- 1ファイル1テーマ。各節に必ずURL。末尾に「未確認・要追跡」節。日本語。
- サイトに公開する場合は、**採り入れた／採らなかった／まだ手当てできていない／未確認**の
  4分類を保つ。**「まだ手当てできていない」を落とした時点で、この班は自己弁護の道具になる。**
- 研究の**良し悪しを論じない**。「この設計にどう使ったか」だけを書く。
  「不採用」ではなく「採らなかった手法」と書く（研究の格付けをしないため）。
