# archive/ — 集計スナップショットの保管庫

将来の研究のために、集計結果を**その時点の設問・判定基準ごと**保存します。

## なぜ回答だけでは足りないのか

各党の立場（stance）は、新しい採決や公約の確認によって**変わります**。
実際に、参政党の「日米同盟・安保協力」の判定は、第221回国会で会派を結成し
日比ACSAに賛成したことを受けて、中立から賛成へ変更しました。

判定基準を保存せずに回答だけ残すと、**後から見た人には数字の意味が復元できません**。
そのため、スナップショットには設問・stance・その根拠まで含めます。

## 更新のリズム

設問は**毎週ではなく、国会の会期ごと**に見直します。
毎週変えると経年比較ができなくなり、研究資料としての価値が失われるためです。
設問は2層構成です。

- **コア設問**：長期に問い続ける争点。原則として文言を変えません。
- **時事設問**：その時期の争点。少数にとどめ、「時事」と明示します。

## ファイル形式

`archive/YYYY-MM.json`

```jsonc
{
  "captured_at": "2026-07-20T18:00:00+09:00",
  "session": "第221回国会",          // 掲載データの会期
  "responses": 4,                     // 回答総数
  "questions": [
    {
      "q": "消費税は引き下げる（または廃止する）べきだ",
      "kind": "core",                 // core | topical
      "basis": "該当する記名採決が無いため、各党の公約と国会での発言にもとづく判定です。",
      "basis_url": "",
      "stance": { "自由民主党": -1, "立憲民主党": 0 }   // +1賛成 / 0中立 / -1反対
    }
  ],
  "aggregate": {
    "p0_agree": 2, "p0_neutral": 1, "p0_oppose": 1,   // 各設問の賛否
    "oi_jimin": 3,                                     // ワンイシューの選択
    "wt0": 1                                           // ◎重視された争点
  },
  "caveat": "自己選択サンプルであり、世論調査ではありません。"
}
```

## 取得のしかた

「みんなの結果」ページの下部にある**スナップショットを保存**ボタンを押すと、
上記の形式の JSON がダウンロードされます。それを `archive/` に置いてコミットしてください。

※ App Check を適用しているため、認証のないサーバー（GitHub Actions 等）からは
Firestore を読めません。自動化する場合はサービスアカウント鍵が必要になるため、
現時点では**ブラウザからの手動取得**としています。

## 選挙期間中の扱い

`config.json` の `election_mode` を `true` にすると、集計ページとアーカイブの公開を停止します。
公職選挙法第138条の3（人気投票の経過・結果の公表禁止）に抵触する恐れがあるためです。
